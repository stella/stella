# Workflows, an agent-native surface, and deeper document review

Stella 0.6 introduces Workflows: multi-step AI runs with explicit review
gates, so longer-running work stays inspectable and approvable at every step.

The MCP and CLI surface is now fully agent-native. A single capability
catalog generates the MCP tools, CLI commands, and skills together; tools
declare idempotence and open-world annotations; agent registration follows
the auth.md protocol; and API/CLI compatibility is enforced at release time.

Document review grows a DOCX suggestion-review flow, exact-passage citation
highlights, document comparison with redlining, and operation undo. Reports
gain export history and completion notifications, and styling gains reusable
style sets with a visual editor.

Accounts can enable two-factor authentication (TOTP with backup codes), and
administrators can inspect user activity logs. Localization adds Indian date
and lakhs/crores number formatting; business registries add Mexico (DENUE).

Under the hood, corpus indexing and ingestion I/O are wall-clock bounded end
to end, and search, rendering, and route performance guards were tightened
across the board.
