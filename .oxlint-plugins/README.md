# Custom oxlint plugins

Project-specific lint rules that catch bugs the general TypeScript / oxlint
ruleset cannot. Each rule encodes an architectural or security invariant of the
Stella codebase. Open the source file for the full pattern logic and the safe /
flagged examples it documents.

Rules are wired up in the root `oxlint.config.ts`. Most plugins replace older
`scripts/lint-*.sh` shell checks; the comment in the plugin source notes the
predecessor when relevant.

## Rules

| Rule                                                                          | Prevents                                                                                                                                                                                                                                                                                                                                                               | Required workaround                                                                                                                                                                                                                                                 |
| ----------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`no-body-ownership-ids`](./no-body-ownership-ids.ts)                         | Cross-tenant data access via `body.workspaceId` / `query.organizationId`.                                                                                                                                                                                                                                                                                              | Source ownership IDs from `validateWorkspaceAccess` (`SafeId`) or `ctx.session.activeOrganizationId`.                                                                                                                                                               |
| [`no-crypto-random-uuid`](./no-crypto-random-uuid.ts)                         | Backend code using Web Crypto UUID generation instead of the Bun-native UUID generator.                                                                                                                                                                                                                                                                                | `Bun.randomUUIDv7()` in Bun-runtime code.                                                                                                                                                                                                                           |
| [`forbid-process-env-outside-env-ts`](./forbid-process-env-outside-env-ts.ts) | Unvalidated `process.env` reads outside env/config/test/script boundaries and approved runtime boundary files.                                                                                                                                                                                                                                                         | Add the variable to the relevant env module, pass it in explicitly, or list a narrow boundary file in `oxlint.config.ts`.                                                                                                                                           |
| [`no-unbranded-ownership-id-param`](./no-unbranded-ownership-id-param.ts)     | Ownership IDs flowing through API parameters as plain strings after validation.                                                                                                                                                                                                                                                                                        | Annotate direct and destructured ownership IDs as `SafeId<"workspace">` / `SafeId<"organization">`, or source them from safe handler context.                                                                                                                       |
| [`no-inline-style-colors`](./no-inline-style-colors.ts)                       | Hex / rgb / named colours hardcoded in `style={{…}}` props that break dark mode.                                                                                                                                                                                                                                                                                       | Use CSS variables (`var(--color-…)`) inside `style`, or a semantic Tailwind utility.                                                                                                                                                                                |
| [`no-nanoid`](./no-nanoid.ts)                                                 | Re-introducing the removed `nanoid` dependency.                                                                                                                                                                                                                                                                                                                        | `Bun.randomUUIDv7()` for IDs; `crypto.getRandomValues()` for custom alphabets.                                                                                                                                                                                      |
| [`no-physical-properties`](./no-physical-properties.ts)                       | Physical Tailwind utilities (`ml-`, `pr-`, `text-left`, `border-l`, `rounded-r`, `left-*`) that ignore RTL.                                                                                                                                                                                                                                                            | Logical equivalents: `ms-`, `pe-`, `text-start`, `border-s`, `rounded-e`, `start-*`.                                                                                                                                                                                |
| [`no-centered-scroll-column`](./no-centered-scroll-column.ts)                 | A centered, width-capped column (`mx-auto` + `max-w-*`) that also owns the vertical scroll (`overflow-y-auto`/`scroll`), so the scrollbar floats at the column edge instead of the content-pane edge next to the inspector rail. (`absolute`/`fixed`/`max-h-*` boxes are exempt.)                                                                                      | Move overflow to a full-width parent (`flex-1 overflow-y-auto`) and keep `mx-auto`/`max-w-*` on an inner content wrapper.                                                                                                                                           |
| [`no-raw-api-url`](./no-raw-api-url.ts)                                       | Hand-written API URLs in `fetch` / `new Request` / `new URL` — relative `/api/...` paths that hit the SPA fallback, and `${env.VITE_API_URL}/v1/...` strings that re-type the origin and version prefix.                                                                                                                                                               | `apiUrl(`/path`)` from `@/lib/api-url`, or the Eden client (`@/lib/api`).                                                                                                                                                                                           |
| [`no-raw-colors`](./no-raw-colors.ts)                                         | Palette Tailwind classes (`bg-stone-50`, `text-gray-900`, `bg-white`) that look unreadable in the opposite theme.                                                                                                                                                                                                                                                      | Semantic tokens: `bg-background`, `bg-muted`, `text-foreground`, …                                                                                                                                                                                                  |
| [`no-raw-foreground-opacity`](./no-raw-foreground-opacity.ts)                 | Foreground opacity utilities (`text-muted-foreground/60`, `placeholder:text-muted-foreground/64`, `bg-muted-foreground/50`) that hide visual intent.                                                                                                                                                                                                                   | Use named foreground attenuation tokens: `text-foreground-muted`, `placeholder:text-foreground-placeholder`, `text-foreground-label`, …                                                                                                                             |
| [`no-raw-error-logging`](./no-raw-error-logging.ts)                           | Raw production error messages, stacks, causes, or stringified errors entering API logs or worker stderr.                                                                                                                                                                                                                                                               | Log `errorTag(error)` as `"error.type"` plus safe correlation IDs.                                                                                                                                                                                                  |
| [`no-secret-in-log-sink`](./no-secret-in-log-sink.ts)                         | Secret-named identifiers/properties (`apiKey`, `clientSecret`, `refreshToken`, `accessToken`, …) reaching `JSON.stringify`, analytics, Sentry, or `new …Error` constructors.                                                                                                                                                                                           | Strip the field, mask it (`maskApiKey`), or use a structural error tag. Inline disable with a `// SAFETY:` comment when the value crosses fetch/URL bodies rather than a logging path.                                                                              |
| [`no-raw-route-query-client`](./no-raw-route-query-client.ts)                 | Direct `queryClient.ensureQueryData` / `ensureInfiniteQueryData` / `fetchQuery` / `prefetchQuery`, or generic query wrappers, inside `beforeLoad` / `loader`, which bypass the route freshness contract; TanStack Query subscription hooks inside route `pendingComponent`s, where abandoned pending renders can receive async updates before commit.                  | `ensureRouteQueryData(...)`, `ensureRouteInfiniteQueryData(...)`, `fetchRouteQuery(...)`, or `prefetchRouteQuery(...)` in route loaders. In pending components, read existing cache synchronously with `useQueryClient().getQueryData(...)`.                        |
| [`no-component-on-redirect-route`](./no-component-on-redirect-route.ts)       | Redirect-only routes that also define `component` / `pendingComponent`, allowing abandoned render trees to mount query subscriptions before the redirect commits.                                                                                                                                                                                                      | Keep unconditional redirect routes inert: only `beforeLoad` with `throw redirect(...)`. Add render components only when redirecting is conditional.                                                                                                                 |
| [`no-ref-mirror`](./no-ref-mirror.ts)                                         | Render-body ref mirrors (`const valueRef = useRef(value); valueRef.current = value`) used to dodge stale closures in effect-installed callbacks.                                                                                                                                                                                                                       | Use `useEffectEvent` for callbacks installed by effects, or derive/read the value directly during render.                                                                                                                                                           |
| [`no-shared-suspense-query`](./no-shared-suspense-query.ts)                   | `useSuspenseQuery` in shared chrome where a cache miss suspends persistent UI.                                                                                                                                                                                                                                                                                         | Use `useQuery` with an explicit loading/disabled state, or keep `useSuspenseQuery` in route/page content backed by loader prefetch or a local Suspense boundary.                                                                                                    |
| [`no-untranslated-jsx-literal`](./no-untranslated-jsx-literal.ts)             | Raw user-facing JSX text that bypasses `use-intl` locale files.                                                                                                                                                                                                                                                                                                        | Render copy through `useTranslations()` / `t(...)`, or use a narrow disable for non-user-facing text.                                                                                                                                                               |
| [`no-untyped-updates`](./no-untyped-updates.ts)                               | `Record<string, unknown>` partial-update objects leaking unknown keys into Drizzle `.set()`.                                                                                                                                                                                                                                                                           | `pickDefined()` from `lib/pick-defined.ts` — returns `Partial<Pick<T, K>>` and catches typos at compile time.                                                                                                                                                       |
| [`mcp-security`](./mcp-security.ts)                                           | MCP OAuth registration secrets in JSONB and ad hoc OAuth client joins that bypass the typed MCP connection loader.                                                                                                                                                                                                                                                     | Redact DCR responses before persistence, and load OAuth client rows through the chat-time MCP connection loader.                                                                                                                                                    |
| [`auth-lifecycle`](./auth-lifecycle.ts)                                       | Partial cleanup when organization membership changes leave auth artifacts on separate lifecycle paths.                                                                                                                                                                                                                                                                 | Route org-member auth cleanup through `revokeOrganizationMemberAuthArtifacts(...)`.                                                                                                                                                                                 |
| [`require-fetch-timeout`](./require-fetch-timeout.ts)                         | `fetch()` calls without a `signal` option, so an upstream hang stalls the handler indefinitely.                                                                                                                                                                                                                                                                        | Pass `signal: AbortSignal.timeout(ms)` (10s for small JSON, 60s for downloads where body streaming dominates), or propagate an upstream signal (e.g. TanStack Query's `{ signal }`).                                                                                |
| [`require-escape-like`](./require-escape-like.ts)                             | Un-escaped interpolations in a Drizzle `like`/`ilike`/`notLike`/`notIlike` pattern: a typed `%`/`_` acts as a wildcard, so `q = "%"` matches every row (and disables search pagination).                                                                                                                                                                              | Wrap interpolated values with `escapeLike()` from `@/api/lib/escape-like` (e.g. `` `%${escapeLike(q)}%` ``); disable with a `// SAFETY:` note when the value is provably wildcard-free.                                                                              |
| [`require-query-limit`](./require-query-limit.ts)                             | Unbounded Drizzle list reads — a `findMany` with no `limit`, or an ordered `select` chain with no `.limit()` — that grow into a memory / latency cliff as a table fills.                                                                                                                                                                                               | Add `limit: LIMITS.x` / `.limit(LIMITS.x)` (apps/api/src/lib/limits.ts) or cursor-paginate; disable with a `// SAFETY:` note when the set is already capped on the write path.                                                                                      |
| [`require-matter-affordance`](./require-matter-affordance.ts)                 | Matter links that silently choose between a browsable listing and an inline reference without the shared matter actions.                                                                                                                                                                                                                                               | Use `<MatterContextMenu>` / `useMatterContextMenu` for listing rows and cards; use `<MatterRefLink>` for inline references.                                                                                                                                         |
| [`require-safe-route-handlers`](./require-safe-route-handlers.ts)             | Authenticated API route files mounting raw inline callbacks or helper functions instead of safe handler endpoint objects.                                                                                                                                                                                                                                              | Move schemas/permissions into a `createSafeHandler` / `createSafeRootHandler` endpoint and mount `endpoint.handler`; disable only for documented protocol/public/dev/streaming exceptions.                                                                          |
| [`require-router-select`](./require-router-select.ts)                         | `useParams` / `useSearch` / `useRouteContext` (or `Route.useParams()` style) called without `select`, subscribing the component to the entire object and rerendering on every change.                                                                                                                                                                                  | Always pass `{ select: (x) => x.field }`.                                                                                                                                                                                                                           |
| [`security-guards`](./security-guards.ts)                                     | Three sub-rules — see below.                                                                                                                                                                                                                                                                                                                                           | Per sub-rule.                                                                                                                                                                                                                                                       |
| `stella-lowercase` from `@stll/oxlint-config`                                 | Mid-sentence `Stella` in string literals, template quasis, and JSX text — the product wordmark is lowercase.                                                                                                                                                                                                                                                           | Lowercase to `stella`. Capital `Stella` is allowed at sentence start (string start or after `. ! ? …`), in PascalCase identifiers (`StellaMark`), kebab asset names, and bundle IDs (`Stella.app`).                                                                 |
| [`folio-layer-boundaries`](./folio-layer-boundaries.ts)                       | Cross-layer imports between folio's render pipeline (`layout-painter` → `layout-bridge`, `layout-bridge` → `layout-painter`, `layout-engine` → either). The painter must stay downstream-only.                                                                                                                                                                         | Move shared types/constants into `layout-engine/types`; move shared measurement helpers into `layout-engine/measure`. Painter may reach the engine only via those two seams. Defence-in-depth test at `packages/folio/src/core/__tests__/layer-boundaries.test.ts`. |
| [`suppression-hygiene`](./suppression-hygiene.ts)                             | Two rules. `require-description`: bare `eslint-disable` / `oxlint-disable` directives with no rationale. `no-foreign-directive`: `biome-ignore` / `prettier-ignore` comments, which this toolchain (oxlint + oxfmt) never honours. (`@ts-ignore` / `@ts-expect-error` are covered by native `typescript/ban-ts-comment`.)                                              | Add a `-- <reason>` trailer to the directive, or a comment on the line directly above (the `// SAFETY:` convention). Remove foreign directives; use an oxlint disable or oxfmt formatting instead.                                                                  |
| [`no-coerced-optional-union-enum`](./no-coerced-optional-union-enum.ts)       | `t.Optional(t.UnionEnum([...]))` in Elysia schemas. Elysia coerces an absent optional `UnionEnum` field to its FIRST member instead of `undefined`, so a handler treating absence as "no value / no filter / use a default" silently gets the first member (mis-scoped query, dead `?? default`). Pinned by `apps/api/src/lib/elysia-optional-union-coercion.test.ts`. | Use `t.Optional(t.Union([t.Literal(...)]))` (does not coerce; absent -> `undefined`) and apply any default explicitly in the handler.                                                                                                                               |
| [`tagged-error-requires-message`](./tagged-error-requires-message.ts)         | A `TaggedError("X")<{...}>()` subclass whose props literal omits `message: string`. Without it the toast / reporting / serialize paths have no human-readable text.                                                                                                                                                                                                    | Add `message: string` to the props type literal and supply it at every construction site.                                                                                                                                                                           |
| [`require-custom-jsonb-column`](./require-custom-jsonb-column.ts)             | Stock `p.jsonb()` / pg-core `jsonb` in Drizzle schemas. The bun-sql driver stores stock JSONB writes as a JSON-string primitive (`jsonb_typeof = 'string'`) instead of parsed JSON. Pinned by the stock-jsonb canary in `apps/api/src/db/columns.test.ts`.                                                                                                             | Use the custom `jsonb` from `@/api/db/columns`, which routes writes through `::text::jsonb`.                                                                                                                                                                        |
| [`no-spread-input-in-query-key`](./no-spread-input-in-query-key.ts)           | Spreading a bare input object/identifier (`...input`, `...filters`) into a TanStack `queryKey` array (or a `*Keys` factory's key array) leaks caller properties into the cache identity -> spurious refetches, stale data, unbounded cache growth.                                                                                                                     | Spread only `*Keys.*()` composition calls; list the cache-identity fields explicitly (`[...entitiesKeys.all(ws), { filters, sorts }]`).                                                                                                                             |
| [`no-unsafe-inner-html`](./no-unsafe-inner-html.ts)                           | Injecting an unsanitized value into `dangerouslySetInnerHTML={{ __html }}` or `el.innerHTML = ...` -> stored XSS. (`react/no-danger` is off; this is its precise, sanitizer-aware replacement.)                                                                                                                                                                        | Pass a string literal, the result of an HTML/DOM escaper or sanitizer (`escapeHtml`, `escapeXml`, `sanitizeHtml`, `DOMPurify.sanitize`), or add a `// safe-html: <reason naming the escaper/source>` comment on the line directly above a verified-safe sink.       |
| [`no-static-devtools-import`](./no-static-devtools-import.ts)                 | TanStack devtools packages or local devtools panels statically imported into ordinary route/component shells, letting devtools code run before the page needs it.                                                                                                                                                                                                      | Import TanStack devtools packages only from the approved lazy-loaded islands; load local devtools modules with `import(...)` behind the devtools toggle.                                                                                                            |
| [`no-static-catalogue-route-import`](./no-static-catalogue-route-import.ts)   | The `/knowledge/tools` route statically importing the heavy catalogue browser and mounting its route-local query subscriptions before the route shell commits.                                                                                                                                                                                                         | Use `import type` for erased catalogue types; keep catalogue browser value imports inside the `React.lazy` import callback.                                                                                                                                         |

### `security-guards` sub-rules

| Sub-rule                 | Prevents                                                                                                                                                                                                                                                                                 | Required workaround                                                                                                                                                                                                                                                                      |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `no-raw-filename-write`  | Path traversal / control chars from `file.name`, `body.fileName`, `part.filename` reaching storage.                                                                                                                                                                                      | Wrap with `sanitizeFilename()` before assigning to a `fileName` property.                                                                                                                                                                                                                |
| `no-unsanitized-href`    | `javascript:` XSS via `<a href={data.url}>` where `data.url` comes from external JSON.                                                                                                                                                                                                   | Wrap with `sanitizeHref()` from `apps/web/src/lib/sanitize-href.ts` (returns `string \| undefined` — `undefined` when unsafe). For backend code that needs structural enforcement, see `sanitizeUrl()` in `apps/api/src/lib/sanitize-url.ts`, which returns the branded `SafeHref` type. |
| `no-unscoped-user-query` | Imports of the `user` table from `auth-schema` without also importing `member` — a heuristic that the query may not be organization-scoped. The rule does not verify the actual join, so reviewers must still confirm that `user` is filtered through a `member` join in the query body. | Import `member` alongside `user`, then join on `organizationId` in the query.                                                                                                                                                                                                            |

## Adding a new rule

1. Drop a new `*.ts` file in this directory exporting `{ meta, rules }` shaped
   like the existing plugins.
2. Register the plugin file in `oxlint.config.ts` under `jsPlugins`. Then enable
   the rule(s) using the `<plugin-name>/<rule-name>` form in the `rules` object
   of `oxlint.config.ts`.
3. Add a row above describing what it prevents and the required workaround.
4. The plugin file's leading comment should state the bug class, list the safe
   patterns it allows, and list the patterns it flags. Existing files are good
   templates.
